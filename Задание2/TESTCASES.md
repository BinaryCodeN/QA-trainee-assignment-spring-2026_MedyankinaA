# TESТCASES.md - Тест-кейсы для Avito QA Internship API

## Базовый URL
`https://qa-internship.avito.com`

## Эндпоинты
| # | Метод | Путь | Описание |
|---|-------|------|----------|
| 1 | POST | `/api/1/item` | Создать объявление |
| 2 | GET | `/api/1/item/:id` | Получить объявление по ID |
| 3 | GET | `/api/1/:sellerID/item` | Получить все объявления продавца |
| 4 | GET | `/api/1/statistic/:id` | Получить статистику объявления |

---

# ТК-001: POST /api/1/item - Создание объявления
# Позитивные
### ТК-001-01 - Создание объявления с валидными данными
- **Предусловие:** сервис доступен
- **Шаги:**
  1. POST `/api/1/item` с телом `{"sellerID": 123456, "name": "Test item", "price": 1000, "statistics": {"likes": 0, "viewCount": 0, "contacts": 0}}`
- **Ожидаемый результат:** HTTP 200, тело содержит `id` (строка UUID), `sellerId`, `name`, `price`, `statistics`, `createdAt`

### ТК-001-02 - Создание нескольких объявлений с одинаковыми полями
- **Шаги:** Отправить два одинаковых POST-запроса
- **Ожидаемый результат:** оба возвращают HTTP 200, `id` у объявлений разные (уникальность ID)

### ТК-001-03 - Создание объявления с price = 0
- **Шаги:** POST с `"price": 0`
- **Ожидаемый результат:** HTTP 200, `price` = 0

### ТК-001-04 - Создание объявления с максимально длинным name
- **Шаги:** POST с `name` длиной 255 символов
- **Ожидаемый результат:** HTTP 200

### ТК-001-05 - statistics содержит нулевые значения
- **Шаги:** POST с `likes=0, viewCount=0, contacts=0`
- **Ожидаемый результат:** HTTP 200, статистика сохранена корректно

# Негативные
### ТК-001-06 - Отсутствует обязательное поле name
- **Шаги:** POST без поля `name`
- **Ожидаемый результат:** HTTP 400

### ТК-001-07 - Отсутствует обязательное поле sellerID
- **Шаги:** POST без поля `sellerID`
- **Ожидаемый результат:** HTTP 400

### ТК-001-08 - Отсутствует обязательное поле price
- **Шаги:** POST без поля `price`
- **Ожидаемый результат:** HTTP 400

### ТК-001-09 - price - отрицательное число
- **Шаги:** POST с `"price": -1`
- **Ожидаемый результат:** HTTP 400

### ТК-001-10 - price - строка вместо числа
- **Шаги:** POST с `"price": "дорого"`
- **Ожидаемый результат:** HTTP 400

### ТК-001-11 - sellerID - строка вместо числа
- **Шаги:** POST с `"sellerID": "abc"`
- **Ожидаемый результат:** HTTP 400

### ТК-001-12 - Пустое тело запроса
- **Шаги:** POST с пустым телом `{}`
- **Ожидаемый результат:** HTTP 400

### ТК-001-13 - name - пустая строка
- **Шаги:** POST с `"name": ""`
- **Ожидаемый результат:** HTTP 400

# Корнер-кейсы
### ТК-001-14 - sellerID на границе допустимого диапазона (111111)
- **Шаги:** POST с `"sellerID": 111111`
- **Ожидаемый результат:** HTTP 200

### ТК-001-15 - sellerID на границе допустимого диапазона (999999)
- **Шаги:** POST с `"sellerID": 999999`
- **Ожидаемый результат:** HTTP 200

### ТК-001-16 - Идемпотентность: повторный POST с теми же данными создаёт новое объявление
- **Шаги:** Отправить одинаковый POST дважды, сравнить `id`
- **Ожидаемый результат:** разные `id` - POST не идемпотентен

# Нефункциональный
### ТК-001-17 - Content-Type ответа
- **Ожидаемый результат:** заголовок `Content-Type: application/json`

### ТК-001-18 - Время ответа при создании объявления
- **Ожидаемый результат:** ответ получен менее чем за 2000 мс

# ТК-002: GET /api/1/item/:id - Получение объявления по ID

# Позитивные
### ТК-002-01 - Получение существующего объявления
- **Предусловие:** объявление создано
- **Шаги:** GET `/api/1/item/{id}`
- **Ожидаемый результат:** HTTP 200, поля `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt` присутствуют и соответствуют созданным данным

### ТК-002-02 - Структура ответа совпадает с созданными данными
- **Шаги:** создать объявление, получить по ID, сравнить поля
- **Ожидаемый результат:** все поля совпадают

# Негативные
### ТК-002-03 - Несуществующий ID (валидный формат UUID)
- **Шаги:** GET с `id = "00000000-0000-0000-0000-000000000000"`
- **Ожидаемый результат:** HTTP 404

### ТК-002-04 - ID - произвольная строка (не валидный формат)
- **Шаги:** GET с `id = "not-a-valid-id"`
- **Ожидаемый результат:** HTTP 400 или 404

# Корнер-кейсы
### ТК-002-05 - Повторный GET одного и того же ID (идемпотентность)
- **Шаги:** GET `/api/1/item/{id}` дважды
- **Ожидаемый результат:** оба запроса возвращают одинаковый ответ HTTP 200

# Нефункциональный
### ТК-002-06 - Время ответа
- **Ожидаемый результат:** ответ получен менее чем за 2000 мс

# ТК-003: GET /api/1/:sellerID/item - Получение всех объявлений продавца

# Позитивные
### ТК-003-01 - Получение объявлений существующего продавца
- **Предусловие:** для sellerId создано ≥1 объявление
- **Шаги:** GET `/api/1/{sellerID}/item`
- **Ожидаемый результат:** HTTP 200, массив объявлений, все с совпадающим `sellerId`

### ТК-003-02 - Все объявления в ответе принадлежат запрошенному продавцу
- **Шаги:** создать 2 объявления с одним sellerID, запросить список
- **Ожидаемый результат:** оба присутствуют в ответе, `sellerId` совпадает у каждого

### ТК-003-03 - Продавец без объявлений
- **Шаги:** GET с sellerID, для которого объявлений нет
- **Ожидаемый результат:** HTTP 200, пустой массив `[]`

# ТК-004: GET /api/1/statistic/:id - Получение статистики

# Позитивные
### ТК-004-01 - Получение статистики существующего объявления
- **Предусловие:** объявление создано с известной статистикой
- **Шаги:** GET `/api/1/statistic/{id}`
- **Ожидаемый результат:** HTTP 200, ответ содержит `likes`, `viewCount`, `contacts`

### ТК-004-02 - Статистика соответствует данным при создании
- **Шаги:** создать объявление с `likes=5, viewCount=10, contacts=3`, получить статистику
- **Ожидаемый результат:** значения совпадают

# Негативные
### ТК-004-03 - Несуществующий ID
- **Шаги:** GET `/api/1/statistic/00000000-0000-0000-0000-000000000000`
- **Ожидаемый результат:** HTTP 404

### ТК-004-04 - ID - невалидная строка
- **Шаги:** GET `/api/1/statistic/invalid`
- **Ожидаемый результат:** HTTP 400 или 404

# Корнер-кейсы

- **Ожидаемый результат:** одинаковые ответы

# E2E-тестирование

### E2E-01 - Полный жизненный цикл объявления
1. POST `/api/1/item` → получить `id`
2. GET `/api/1/item/{id}` → проверить данные
3. GET `/api/1/{sellerID}/item` → объявление есть в списке
4. GET `/api/1/statistic/{id}` → статистика совпадает

### E2E-02 - Изоляция данных продавцов
1. Создать объявление для sellerID_A
2. Создать объявление для sellerID_B
3. GET `/api/1/{sellerID_A}/item` → только объявления A
4. GET `/api/1/{sellerID_B}/item` → только объявления B

### E2E-03 - Множественные объявления одного продавца
1. Создать N объявлений с одним sellerID
2. GET `/api/1/{sellerID}/item` → количество равно N, все ID уникальны
